How the Algorithm Works:

Step 1: Setup

We start with a player position and a ray direction. The algorithm needs to find where this ray first hits a wall.

pos_x = player_x / TILE_SIZE
pos_y = player_y / TILE_SIZE

Step 2: Calculate Delta Distances

This is the key insight! Instead of checking every tiny point along the ray, we calculate how far the ray travels between grid lines.

delta_dist_x = |1 / ray_dir_x|
delta_dist_y = |1 / ray_dir_y|

Why this works: If the ray direction is (0.5, 0.8), then delta_dist_x = 2.0 means the ray travels 2 units of distance for every 1 unit it moves in the X direction.

Step 3: Determine Step Direction

Figure out which direction we're stepping in the grid (+1 or -1) and calculate the initial distance to the next grid line.

if (ray_dir_x < 0): step_x = -1, side_dist_x = (pos_x - map_x) * delta_dist_x
else: step_x = 1, side_dist_x = (map_x + 1 - pos_x) * delta_dist_x

Step 4: DDA (Digital Differential Analyzer)

This is where the magic happens! We "step" along the ray by jumping from grid line to grid line, always choosing the closest one.

while (not hit):
  if (side_dist_x < side_dist_y):
    step in X direction
  else:
    step in Y direction
  check if we hit a wall

The genius: We never miss a wall because we check every grid cell the ray passes through, but we do it efficiently by jumping between grid lines!

Step 5: Calculate Final Distance

Once we hit a wall, we calculate the exact distance from the player to the wall intersection point.

if (side == 0): wall_dist = (map_x - pos_x + (1 - step_x) / 2) / ray_dir_x
else: wall_dist = (map_y - pos_y + (1 - step_y) / 2) / ray_dir_y

Key Insights:

  • Grid-based stepping: Instead of checking every pixel, we jump efficiently between grid lines
  • Delta distances: Pre-calculate how far the ray travels per grid step
  • Side tracking: Keep track of whether we hit a vertical or horizontal wall (affects lighting in 3D)
  • Exact intersection: Calculate the precise distance to create accurate 3D projections
